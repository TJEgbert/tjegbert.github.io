Update 9/2/2024

Wow it's been a while since I have updated this. For a quick recap passed all my class for spring and summer with flying colors! Also, I ended up doing an internship with Autoliv as IT during the summer! Those are two big things that have prevented me from working on projects and or updating the website. Now with the internship over and fall semester not seeming too bad hopefully I can work on some projects and let everyone know about them. I do have an update for the turn-based RPG that I've been working on though!

Summer classes weren't too bad, I had software development two which is a class with a lot of group work. So, to say I was very concerned about that is an understatement, but surprisingly it went well. There were communication issues between the group, but I made the extra effort and essentially took the minutes when we would have a group call. Then once we were done, I would have everyone on the call verify it was correct or add anything that I might have missed. Then I would upload that into the group chat so if anyone missed the meeting they could still be informed. My other classes operating systems, and Apple app development went well. With the Apple class I did learn how to code Swift.

I was fortunate enough to get an IT internship with Autoliv last summer. When I heard the term IT my mind like most jumps to telling people to turn off and on their computer again. Don't get me wrong that did happen, and it fixed most issues that people had when the computer wasn't running quite right. At least from Autoliv there is so much more that IT handled than that. With us being admins for the sight we had our hands in a lot of places. Some of the things we would have to approve are software, user access to folders on the network drive, and giving users admin access over a certain computer so they can install department specific software. We also managed the inventory of most of the electronics in the building. This being computers, monitors, keyboards, mice, PCI cards, every type of cable and so much more. There were some electronics we did not manage and that being anything to do with the automation machines. Most of the production that went on at my facility was automated so that was handled by a different team. As local IT we still had to help with the automation to make sure their connection to the network is working and if something was wrong with that, we would work with other teams to get it fixed. One project that started while I was there was fixing printing issues in bay four. The ticket came in saying there was a network issue in bay four, the full time IT and I went and checked what was wrong. We pinged the devices and they all seem to be online. We went out to one of the devices with one of the automation engineers to check their uplink with the network fluke to make sure our uplink was good. It was so we reached out to our network team to see if they had any ideas what could be wrong. They noticed there was a lot of traffic on that switch, more than what should have been. Which come to find out there were unmanaged switched on each machine which was throwing a lot of back traffic onto that switch stack. We tried to run a new cable from a different switch stack to one of the machines. This improved the situation but didn't fix it. So, we ended up putting configuration on the unmanaged switches so hopefully permanently fix the problem. This project ended up being put on hold due to the cost of new switches since the older ones were going to be life cycled in the following year. I learned a lot about how switches and networks work in an industry setting thanks to this project and what the full time IT person taught me.

Two projects that we got done while I was there were switching out the last Windows 7 computer in the test lab that did not have a TPM chip. We set up a computer on Windows 10 LTSC and got the needed software installed with help from one of the global teams. When we went to swap out the computer we were there to help with any hardware-related issues. The global team was in Utah at the time for some training, so they all came out to our facility to help get the software working. It took almost all day, but we got it working at one point I realized the new cable we used wasn't working. I ended up grabbing a the old working one and that fixed a lot of the issues they were having.

The other project we got done was switching all manufacturing computers to Windows 10 LTSC to help avoid down time when the force Windows 11 update happens. Between the full-time IT and I in the time I was there we set up about ten or so computers to switch out. I have experience setting up personal computers but nothing in an industrial setting. Luckly the client team made this process simple to install Windows 10 LTSC with computer standard configuration on the new machines. Then it was our job to get everything else set up and ready. We got the generic account ready so the operators wouldn't have to worry about signing in. We deployed any need software and got any permissions needed on the computer. We then reached out to the supervisor over the area where we were swapping out the computer and set up time, so we could switch it out. Once we got it switched out, we would decommission the old computer software the company used to track the computers.

The last project that unfortunately I could not be a part of, but I learned a lot was VLAN standardization. Since I only being an intern, I did not have network access so I could not change the VLANS on the switch. So, I ended up watching and learning from the full time IT about the VLAN standardization process and why they are so important for security. Thats especially is true for our facility since they are always worried about downtime of the machines. The big one was making it, so the computers of the manufacturing machine weren't on the same VLAN as the office computers. This protects networks since if a virus is downloaded on one network it can't spread easily to the other. To be honest with how locked down the internet is on a manufacturing PC the virus would come from the office staff. So, we were basically protecting the manufacturing machine from the office users. But that was pretty much it for what I worked on while at Autoliv. We also had a project of upgrading any office computer that wouldn't be able to upgrade to Windows 11. This wasn't due till October 2025, but we got it pretty much all the way done while I was there. They also started to roll out the Windows 11 update to office users, so we had a lot of people who needed help when something went wrong with the installation.
