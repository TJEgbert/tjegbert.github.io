Unreal experience

Since I still want to work on video games, I'm learning to work in Unreal Engine 5. I do have an advantage here since they use C++ for their coding. C++ is the language in which I have the most experience in, this is more about learning Unreal Engine specific syntax. For this I'm following along with some tutorials on YouTube. In the last video I was working with there was some code that would not compile for me but would compile for him in the video. After four hours over a couple of days I was finally able to get the code to compiler. It had to do with method implementation in the cpp. I'm not sure why they have to be implanted in different ways. My guess is since the one is an empty method to be used by blueprints to essentially make the method and the other uses variables specific to the class. You can see what I mean in the picture to side of this.

I am working alongside the video Beginners Intro to UE5 - Create a Game in 3 Hours in Unreal Engine 5. Once I'm done with this, I feel I will have the skills needed to make my block puzzle game. I am looking forward to starting this project since I always love a challenge.

Math Game

This was an assignment for CS3280 Object Oriented Windows Applications. I was excited to work on this since it aligns with one of my goals, to work on video games. It is rather a simple game, but it is a game non the less. The goal of the assignment was to make a kids math game that would time them and show how many questions they answered right or wrong. Then it needed a theme to go along with it. I trying to be the cool uncle, made this game for the nieces and nephews who all love Pokémon. So, I decided to go with a Pokémon themed math game. I'm planning on working more on this project on the side so I can flush out more of mechanics so the nieces and nephews can play it as they grow up.
